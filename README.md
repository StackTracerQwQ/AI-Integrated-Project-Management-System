# AI Integrated Project Management System

## Collaboration Guide

This repository is organized by document purpose:
- `docs/`: project scope, deliverables, user stories, and planning context
- `agile/`: sprint planning and Scrum process artifacts
- `tasks/`: executable engineering task specifications

## Document Ownership
- `docs/project_charter.md`: Project Owner / Supervisors
- `docs/deliverables.md`: Project Owner / Supervisors
- `docs/user_stories.md`: Product Owner
- `agile/scrum_framework.md`: Scrum Master
- `agile/sprint_00.md`: Scrum Master (with team input)
- `tasks/`: Engineering team (implementation-ready task specs)
- Repository maintenance (branch strategy, pull request merge, release tagging): Repository Maintainer

## How to Work in This Repository
1. Keep business decisions in `docs/`.
2. Keep process cadence in `agile/`.
3. Keep implementation-ready work items in `tasks/`.
4. Keep acceptance criteria testable and observable.
5. Record assumptions and unresolved dependencies explicitly.
